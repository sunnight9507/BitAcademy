1 대 1 : 두개를 합치든 각 기본키를 붙여주던지
1 대 다 : 1의 기본키를 다 쪽으로
다 대 다 : 독립적인 테이블 하나 생성


- 좌변-최소성
: 우변의 값을 만족시키기 위한 좌변의 값중에 최소성을 만족하는지
ex) 학번 -> 이름 (O)
     학번,주번 -> 이름 (X)

- 이행규칙
: A->B / B->C => A->C

정규화 과정
- 1정규화 : 원자 값
  - 기본키 확인
  - 각 기본키가 결정하는 것 확인(좌변최소성 확인) => 2정규화

- 2정규화 : 좌변최소성 만족
  - 이행적 규칙 유무 확인 => 3정규화

- 3정규화 : 이행적 종속을 삭제
